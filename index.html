RAG, LLM, Embedding, Deep Learning & Machine Learning

European AI Regulatory Compliance Engine (Multi-Agent RAG)

Retrieval-Augmented Generation for AI Regulatory Compliance

Designed and built a production-grade Multi-Agent RAG system that answers nuanced compliance queries across three of the world's most consequential AI governance frameworks: the EU AI Act, GDPR, and the NIST AI Risk Management Framework. The system indexes 280 pages and over one million characters of regulatory text into 2,758 semantic chunks, retrieved through a hybrid pipeline combining Qdrant dense vector search (Gemini embeddings, 768-dim) and BM25 sparse keyword search fused via Reciprocal Rank Fusion.

At the core is a 3-agent agentic pipeline: an Internal Researcher queries the regulatory document store, an External Fact-Checker cross-validates findings against live web sources via DuckDuckGo and FastMCP, and a Synthesizer produces a final grounded response. Every answer is traceable through Sequence Matcher word-level alignment, highlighting the exact source passage in the original PDF. Evaluated against a hand-labeled 20-question benchmark, the system achieved 100% generation accuracy.
RAG LLM Embedding Deep Learning Machine Learning Python Qdrant
MLOps, Data Engineering & Machine Learning

End-to-End MLOps Pipeline: Taxi Fare Prediction

Random Forest model served via FastAPI, Streamlit frontend, PostgreSQL, and Airflow 3 — all orchestrated with Docker

Architected a fully automated, production-grade MLOps system for NYC Taxi Fare prediction that treats the model not as a deliverable but as a living component of a larger, self-sustaining infrastructure. The pipeline covers the entire model lifecycle — from scheduled data ingestion and preprocessing through to deployment, monitoring, and autonomous retraining.

Data integrity is enforced from the first step: Great Expectations validates every incoming batch against defined data contracts, catching schema violations before they propagate downstream. Airflow 3 DAGs orchestrate the entire workflow, triggering retraining when production model performance degrades. A Champion-Challenger strategy benchmarks the retrained model against the current production model on held-out data before any promotion decision is made. All experiments, model versions, parameters, and metrics are tracked in MLflow, and the final model is served through a FastAPI endpoint containerised in Docker.
